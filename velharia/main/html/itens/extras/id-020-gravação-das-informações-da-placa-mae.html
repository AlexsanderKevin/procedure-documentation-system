Gravação das Informações da Placa-Mãe

Descrição

É possível que o reparo ou a substituição da Placa-Mãe apague da BIOS as informações de SN, MTM, modelo, etc. Sendo assim, é necessário que regravemos estas informações antes de iniciarmos os testes.

Soluções

  • Acessando o UONE

    1. Acesse a BIOS da máquina testada pressionando o botão recovery com o computador desligado, ou pressionando a tecla F2 durante a inicialização.

    2. Uma vez na aba information (informação) da BIOS verifique se as informações de Lenovo SN, MTM e Product Name constam INVALID (inválido).

    3. Verifique se a opção Boot Mode (Modo de Inicialização) está em UEFI na aba Boot (Inicialização).

    4. Na aba Security (segurança) desative a opção Secure Boot (Inicialização Segura).

    5. Conecte o pendrive UONE na porta USB do equipamento.

    6. Salve a configuração e inicie a máquina pressionando F10 e confirmando.

    7. Acesse o Boot Menu (Menu de Inicialização) pressionando F12 durante a inicialização.

    8. No Boot Menu selecione a opção [...]

    9. A máquina inicializará pelo sistema operacional instalado no pendrive.

    10. Com o sistema aberto, pressione a tecla C para selecionar a opção [...] e pressione enter.

    11. Surgirá um terminal de comando, que deverá ser utilizado para a regravação.

  • Gravando as informações

    Uma vez no terminal, cada informação deverá ser gravada com um comando específico mas que começam todos com: uone.nsh

    Os códigos para gravar cada informação são:

    • uone.nsh /ln :para gravar o SN.

    • uone.nsh /mt :para gravar o MTM.

    • uone.nsh /pn :para gravar o modelo da máquina.

    • uone.nsh /UU AUTO :para gravar o UU ID number.

    Na mesma linha em que o código é inserido, deve ser digitada a informação que está sendo gravada.

    Exemplo: uone.nsh /ln PE0AB12

    O modelo deve estar entre parentesis.

    Exemplo: uone.nsh /pn "Lenovo IdeaPad S145-15API"

    O UU ID number não precisa de nenhuma informação além de digitar uone.nsh /UU AUTO

Obs:

  • Após a gravação é recomendável que certifique-se de que as informações foram gravadas corretamente acessando a BIOS

  • Grave as informações antes de fazer o LDIAG, pois essas informações ficam armazenados em um banco de dados para consultas futuras.
